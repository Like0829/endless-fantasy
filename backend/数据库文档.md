# 无尽幻境 — 数据库说明

## 概览

- **数据库名**: `endless_fantasy`
- **字符集**: `utf8mb4`（支持中文和 emoji）
- **引擎**: InnoDB（支持行锁、事务）

---

## 表结构

### 1. world_state — 世界状态

单行热点表，存储全局世界状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 固定为 1，保证单行 |
| atmosphere | INT | 城镇氛围值 0-100，默认 50 |
| updated_at | TIMESTAMP | 自动更新时间戳 |

**特点**：
- 单行设计，所有玩家读取同一个状态
- 原子 UPDATE 避免并发冲突
- `ON UPDATE CURRENT_TIMESTAMP` 自动记录更新时间

**常用 SQL**：
```sql
-- 读取
SELECT atmosphere FROM world_state WHERE id = 1;

-- 更新（+10）
UPDATE world_state SET atmosphere = GREATEST(0, LEAST(100, atmosphere + 10)) WHERE id = 1;
```

---

### 2. tasks — 任务表

存储游戏中的悬赏任务，支持接取和完成。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) (PK) | 任务编号，如 `task_001` |
| name | VARCHAR(200) | 任务名称 |
| description | TEXT | 任务描述 |
| status | ENUM | 状态: `pending` / `in_progress` / `completed` |
| player | VARCHAR(100) | 接取任务的玩家名，NULL=未接取 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**索引**：
| 索引名 | 字段 | 用途 |
|--------|------|------|
| idx_status | status | 查询待接取/进行中的任务 |
| idx_player | player | 查询某玩家的任务 |

**常用 SQL**：
```sql
-- 所有任务
SELECT * FROM tasks ORDER BY created_at;

-- 待接取任务
SELECT * FROM tasks WHERE status = 'pending';

-- 接取任务（行锁防并发）
BEGIN;
SELECT * FROM tasks WHERE id = 'task_001' FOR UPDATE;
UPDATE tasks SET status = 'in_progress', player = '玩家名' WHERE id = 'task_001';
COMMIT;

-- 完成任务
UPDATE tasks SET status = 'completed' WHERE id = 'task_001' AND player = '玩家名';
```

---

### 3. messages — 留言表

玩家可在各个地点留下信息，其他玩家可查看。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK, AUTO_INCREMENT) | 自增主键 |
| location | VARCHAR(100) | 留言地点 |
| content | TEXT | 留言内容 |
| player | VARCHAR(100) | 留言玩家名 |
| player_status | ENUM | 留言时状态: `alive` / `dead` |
| created_at | TIMESTAMP | 留言时间 |

**索引**：
| 索引名 | 字段 | 用途 |
|--------|------|------|
| idx_location | location | 按地点查询留言（最常用） |
| idx_player | player | 按玩家查询留言 |
| idx_created_at | created_at | 按时间排序 |

**常用 SQL**：
```sql
-- 查看某地点的留言（按时间倒序）
SELECT * FROM messages WHERE location = '教堂' ORDER BY created_at DESC;

-- 添加留言
INSERT INTO messages (location, content, player, player_status)
VALUES ('教堂', '这里有问题', '调查员', 'alive');
```

## 并发控制

| 场景 | 策略 |
|------|------|
| 世界状态读取 | 不锁定，直接 SELECT |
| 世界状态更新 | 行锁 `FOR UPDATE` |
| 任务接取 | 行锁 `FOR UPDATE` + 状态检查 |
| 任务完成 | UPDATE + WHERE 条件校验 |
| 留言写入 | 单条 INSERT，无需锁 |
